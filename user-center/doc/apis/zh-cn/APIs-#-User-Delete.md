# 注销当前用户

## 请求

```http
DELETE /user
Authorization: Bearer <access_token>
```

无请求参数, 无请求体. 待删除用户由访问令牌中的 `sub` 声明确定.

## 响应

成功: `204 No Content`, 响应体为空.

### 错误响应

| HTTP 状态 | `error`             | 说明                                             |
|---------|---------------------|------------------------------------------------|
| `401`   | `invalid_token`     | 访问令牌缺失, 无效, 或无法解析到已存在的用户.                       |
| `404`   | `not_found`         | 令牌对应的用户在服务端已不存在.                               |
| `500`   | `server_error`      | 其它内部错误.                                        |

错误响应体格式:

```json
{
  "error": "not_found",
  "error_description": "..."
}
```

## 备注

- 该接口用于用户注销自身账号, 不能删除他人.
- 删除成功后, 已签发的访问令牌在 JWT 过期前可能仍会被网关或资源服务器视为有效, 客户端应在收到 `204` 后立即清理本地会话与令牌.
- 不进行密码或验证码二次确认, 由客户端在 UI 层负责风险提示.
